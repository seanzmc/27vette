# Next Workbook Pass For Rules And Price Overrides

## Summary
The new structure is pointed in the right direction. The immediate next pass should populate the new `rules_import` fields, tighten the `price_rules` schema, and run a sequencing audit against the updated `Section_Master`.

## Key Changes
- Keep the sheet name as lowercase `price_rules` unless you want to standardize names later; the workbook currently has `price_rules`.
- Populate every `rules_import.rule_id` with a deterministic ID: `rule_{source_id}_{rule_type}_{target_id}`, lowercased and sanitized.
- Populate `source_section`, `target_section`, `source_selection_mode`, and `target_selection_mode` from:
  - `stingray_master.option_id -> section_id` for main options.
  - `LT_Interiors.interior_id -> section_id` for LT interior sources.
  - Add `section_id` to `LZ_Interiors` only when LZ interior rules are imported.
- Add a derived audit column or exported review report for rule handling:
  - `drop_redundant` for same-section `excludes` in `single_select_req` or `single_select_opt`.
  - `keep` for cross-section rules and same-section `multi_select_opt` rules.
  - `review` for same-section non-excludes, especially `opt_tvs_001 requires opt_zf1_001`.

## Price Rules

  - `condition_option_id`
  - `target_option_id`
  - `price_rule_type`
  - `price_value`
  - `review_flag`
  - `notes`
- Represent the current TVS case as: `condition_option_id=opt_z51_001`, `target_option_id=opt_tvs_001`, `price_rule_type=override`, `price_value=0`.
- Remove or complete the partial `pr_b6pc_001` row before import, so blank rules do not enter the build contract.

## Requires Sequencing
- Use category order plus section display order as the form order.
- The current updated order still leaves 11 `requires` rules where the required target appears after the source, mostly LPO Exterior requiring Spoiler or Performance selections.
- For those rows, either move the target section earlier or keep the source option visible with a disabled state and a clear required-target message.
- Do not auto-select `requires` targets; `requires` only validates eligibility.

## Test Plan
- Verify all 288 populated `rules_import` rows have nonblank `rule_id`, source/target sections, and source/target selection modes.
- Confirm same-section single-select `excludes` are marked redundant, not imported as active rules.
- Confirm same-section `requires` rows are flagged, not dropped.
- Confirm `price_rules` has no blank partial rows and the Z51 plus TVS override resolves TVS price to `0`.
- Confirm the requires sequencing audit lists every source whose target section appears later in the form order.

## Assumptions
- The workbook remains the source of truth, not a separate generated CSV.
- `rules_import` can carry derived columns directly, even if they are later generated by script/formula.
- The build UI can consume a separate `price_rules` table.
