# Repository File Organization

This repo has active production inputs, generated production artifacts, experimental CSV-shadow artifacts, and archived reference material. A file referenced by a current script or test is active even if a historical copy also exists under `archived/`.

## Active Source Inputs

- `stingray_master.xlsx` is the production source workbook while Excel remains the production source.
- `scripts/generate_stingray_form.py` is the production Stingray generator entrypoint.
- `scripts/corvette_form_generator/model_configs.py` owns model-specific active input paths.
- `architectureAudit/stingray_interiors_refactor.csv` is an active Stingray interior hierarchy input because the generator config and regression test reference it.
- `architectureAudit/grand_sport_interiors_refactor.csv` is an active Grand Sport interior hierarchy input because the generator config references it.
- `data/stingray/**/*.csv` is the experimental Stingray CSV-shadow source package.
- `data/stingray/validation/projected_slice_ownership.csv` is the active CSV-shadow ownership manifest.
- `form-app/index.html`, `form-app/app.js`, and `form-app/styles.css` are the static production app shell.

## Generated Production Artifacts

- `form-app/data.js` is generated by the workbook generator and consumed by the static app.
- `form-output/*` contains generated JSON, CSV, and inspection outputs.
- Generated `form_*` sheets inside `stingray_master.xlsx` are generator output, not hand-authored source.

Do not edit generated production artifacts by hand. If generator validation rewrites these files during a non-output pass, restore the generated churn before finishing unless the pass explicitly approves committing it.

## Experimental Ignored Artifacts

- `build/experimental/form-app/*` is the ignored CSV-shadow app shell.
- Shadow overlay scratch output, such as `/tmp/stingray-shadow-data.json`, is temporary test output.
- `build/experimental/` must remain ignored by git.

Do not copy `build/experimental/form-app/data.js` into production `form-app/data.js`, and do not commit ignored experimental builds.

## Archived And Reference Material

- `archived/docs/*` contains historical docs, old architecture audits, specs, prompts, and reference copies.
- `archived/referenceSheets/*` contains historical reference-sheet material.
- Deprecated planning docs are retained for traceability unless a later cleanup pass explicitly deletes them.

`archived/` means not required for current generator/test gates. In plain terms: archived/ means not required for current generator/test gates. Do not move active inputs into archived paths.

## Deprecated Or Cleanup Candidates

- Historical planning, audit, and debug docs under `archived/` may be cleanup candidates in a future pass.
- Incidental local metadata files, such as `.DS_Store`, may be cleanup candidates if tracked or surfaced by status.

Do not delete cleanup candidates in an organization pass unless deletion is explicitly approved.

## Files That Must Not Move Casually

- `stingray_master.xlsx`
- `scripts/generate_stingray_form.py`
- `scripts/corvette_form_generator/model_configs.py`
- `architectureAudit/stingray_interiors_refactor.csv`
- `architectureAudit/grand_sport_interiors_refactor.csv`
- `form-app/data.js`
- `form-output/*`
- `data/stingray/**/*.csv`
- `data/stingray/validation/projected_slice_ownership.csv`

If a file is referenced by a script or test, update all references before moving it. An archived copy may exist for history, but it must not be the only location of an active required file.

## Rules

- Do not move active inputs into archived paths.
- Do not edit generated production artifacts by hand.
- Do not commit ignored experimental builds.
- If a file is referenced by a script or test, update references before moving it.
- `archived/` means not required for current generator/test gates.
